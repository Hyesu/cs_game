# Most Important Principles
- 모든 피드백은 한국말로 대답한다.
- 사용자의 동의 없이 파일을 삭제하지 않는다.
- 작업 진행에 모호한 부분이 있는 경우 반드시 물어보고, 방향에 대하여 확신이 드는 경우 진행한다.

# 설계 원칙
- TDD를 방법론으로 채택한다.
    - 클래스 기능을 대표하는 빈 메소드를 먼저 만들고, 사용자에게 검토 받는다.
    - 만든 메소드에 대한 테스트를 만들고, 사용자에게 검토 받는다.
    - 기능을 구현한다.
- 설계시 SOLID 원칙을 준수하되, 가장 중요한 것은 Single Responsibility Principle로 생각해야 한다.
